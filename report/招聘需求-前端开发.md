## 💡 项目特点与挑战

### 1. Widget-First 架构
- 每个可视化模块都可独立嵌入到原生 App 的 WebView 中
- 需要设计透明布局、无边距的 Widget 组件
- 支持通过 URL 参数配置语言和主题

### 2. 低性能设备适配
- **目标设备**：低端 Android 平板（如 2GB RAM、4 核 CPU）
- **挑战**：图表渲染卡顿、滚动不流畅、内存占用高
- **要求**：
  - 优化图表数据点数量（数据采样、聚合）
  - 使用虚拟滚动、懒加载技术
  - 减少不必要的重渲染
  - 使用 `useMemo`、`useCallback` 优化性能
  - 图表动画降级处理

### 3. Adapter Pattern 数据转换
- 后端 API 返回中文标签数据（如"周一"、"正常血压"）
- 前端通过 Adapter 层转换为标准化的领域模型
- **禁止**直接在 UI 中显示后端返回的 label 字符串
- **必须**通过 i18n 翻译 key 渲染

### 4. WebView 通信
- 实现与原生 App 的数据交互（如获取用户 Token、接收设备数据）
- 处理 WebView 的生命周期事件
- 适配不同版本的 WebView 容器

## 📝 开发规范

### 代码风格
- 使用 **函数式组件** + **Hooks**，禁用 Class 组件
- 使用 `@/` 作为路径别名（如 `@/components/common`）
- 所有颜色定义在 `src/config/theme.ts` 中统一管理
- 组件文件使用 PascalCase 命名（如 `BPStatisticsCard.tsx`）

### 组件命名规范
- 统计卡片：`[Feature]StatisticsCard`（如 `HRStatisticsCard`）
- 数据分析：`[Feature]DataAnalysisCard`
- 周概览：`[Feature]WeeklyOverviewCard`
- Widget：`[Feature]TrendWidget`

### 性能要求
- 首屏加载时间 < 2 秒（低端设备）
- 图表渲染帧率 ≥ 30 FPS
- Bundle Size 优化（使用代码分割、Tree Shaking）
